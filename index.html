我看到你貼的 v1.4.8 已經加了 CORS Proxy，但出現「網路請求被攔截」還是常見問題，原因是： https://corsproxy.io/ 不穩定，容易被封鎖或限制大小。 前端直接呼叫 Hugging Face API，本質上免費方案容易排隊或超時。 前端瀏覽器安全限制（CORS）很難完全避免，如果 Proxy 失效就會跳錯誤。 建議方案（完全不需要你懂程式碼、直接複製貼上就可用）： 方案 A：前端直接用 FormData，去掉 Proxy（穩定性最高，HF 免費 Token 可用） 我幫你修改去背函數如下，只需要替換你原本 removeBG 這段： const removeBG = async () => { if (!uploadedBlob) return alert("請先選取或拍照"); setIsProcessing(true); try { const endpoint = "https://api-inference.huggingface.co/models/briaai/RMBG-1.4"; const formData = new FormData(); formData.append("file", uploadedBlob); const res = await fetch(endpoint, { method: "POST", headers: { "Authorization": `Bearer ${HF_TOKEN}` }, body: formData }); if (!res.ok) { const text = await res.text(); console.error("API Error:", text); alert("去背失敗，請稍後再試或更換照片"); setIsProcessing(false); return; } const blob = await res.blob(); const url = URL.createObjectURL(blob); const newImg = new Image(); newImg.onload = () => { setImage(newImg); setIsProcessing(false); }; newImg.src = url; } catch(e) { console.error(e); alert("網路或 API 錯誤，請稍後再試"); setIsProcessing(false); } };
